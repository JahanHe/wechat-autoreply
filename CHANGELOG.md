# 更新日志

本文件记录项目的重要变更。

本项目采用务实的更新日志写法，并尽量使用语义化版本标签。

## 未发布

暂无。

## [0.4.6] - 2026-06-13

### 新增

- 新增知识库 > 客户记忆页，支持搜索、展开查看、复制客户 ID、本机压缩和 AI 压缩。
- 新增客户记忆管理 IPC，用于获取统计、单条压缩和批量压缩。
- AI 参考页新增 Markdown 预览卡片和弹窗编辑器。
- 监控总览新增视觉状态卡、回复来源比例条和右侧快照详情。

### 变更

- 知识库二级入口更新为知识总览、规则库、AI 参考、测试中心、客户记忆。
- 规则行、客户记忆行和 AI 参考卡片的高频操作改为图标按钮，并保留 tooltip 和无障碍标签。
- 监控页移除重复的“当前状态”和“快捷操作”板块。
- 本机通知统一带应用 logo。

### 修复

- 保存 AI 参考开关时不再清空已有长文本配置。
- 客户记忆可压缩为长期摘要，降低历史对话过多导致的上下文膨胀。
- 小窗口下客户记忆、AI 参考和监控看板改为单列响应式布局。

## [0.4.5] - 2026-06-13

### 新增

- 新增统一本机知识服务，聚合 knowledge-base、手动知识、参考回复和外部同步缓存。
- 新增知识总览、AI 参考和测试中心。
- 新增 `main-test-reply-pipeline`、知识总览、资料详情和本机检索 IPC。
- 新增生产零远端查询和本机知识索引自动测试。

### 变更

- 外部知识源改为只连接、下载和增量同步，生产回复只检索本机资料。
- Bot 策略、AI API 和外部知识同步移入设置。
- 规则库改为紧凑摘要列表和单条展开编辑。
- 用户可见“判断库增强”统一改为“同步资料增强”。

### 修复

- Cookie 过期或同步服务断开不再影响已有本机资料参与 AI 回复。
- 避免规则页一次渲染全部大型编辑器导致页面冗长和图片预览开销过大。

## [0.4.4] - 2026-06-13

### 新增

- 新增 ReplyTask 回复任务，追踪客户消息从识别、承接、AI API最终回复、完成度检查到补救/转人工的完整链路。
- 新增客户记忆，按会话客户 ID 记录最近消息、客服回复、已发送规则、已发送动作和未完成任务。
- 新增 `/completion/check` 回复完成度检查接口，支持 AI API 检查并在不可用时退化为启发式检查。
- 新增规则候选池，AI API 成功解决的问题会生成待人工审核的规则候选。

### 变更

- 统一术语：“本地 AI 服务”改为“本机回复中转服务”，“AI 回复”明确为“AI API 回复”。
- 规则支持 `final`、`quick_then_api`、`action_only`、`ignore` 四种模式，旧规则默认 `final`。
- 15 秒承接语不再代表最终回复；60 秒默认只标记延迟、写日志和通知，不直接发送固定兜底。
- 回复日志和 Webhook 总结新增任务、客户、规则模式、完成度、补救、延迟和规则候选统计。

### 修复

- 避免承接语、重复 AI 回复或空 AI 回复导致异步最终回复链路被误判为完成。
- 非最终回复会继续进入完成度监管，未完成时按次数限制补救或转人工。

## [0.4.3] - 2026-06-13

### 新增

- macOS DMG 新增 `repair-macos-install.command` 一键修复脚本，用于自动复制或覆盖修复 App、检查安装完整性、清除隔离属性并打开应用。

### 修复

- `doctor` 会同时检查仓库根目录和桌面运行目录中的 `.env`，避免安装后只在运行目录配置时误报缺少密钥或 Webhook。
- 启动时会把运行目录中的 Webhook 配置同步到通知配置；首次发现 Webhook 时自动启用通知，但不覆盖用户之后主动关闭通知的选择。
- 客服页配置存储桥迁移到 `window.wechatKfDesktop.storage`，不再覆盖客服页自身的 `window.chrome`。
- 外部知识库配置窗口检测到新访问凭证后会自动验证，验证和初始化成功后自动关闭配置窗口，并保留已确认的过期、无权限、错误和超时状态。
- 登录二维码截图优先截取已挂载的客服页 `BrowserView` 区域，提高 Webhook 登录截图的准确性。
