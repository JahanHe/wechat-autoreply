## 规则类型

1. 文字规则：只发文本，写在“关键词文字规则 JSON”。
2. 动作规则：可组合文本、图片、商品、素材库等，写在“动作规则 JSON”。
3. 独立图片规则：仅在确实需要“关键词直接发图”时使用。当前默认不用独立图片，图片都作为动作规则里的补充。
4. 忽略规则：客户只是道谢、明白、OK 时，用 `ignore` 动作标记已处理，但不发消息。

## 文字回复限制

文字回复最多发送两次。每次内容可以较长，但不要拆成很多条。

示例：

```json
{
  "enabled": true,
  "name": "会员退款",
  "keywords": ["退款", "退货", "售后"],
  "reply": "7天内支持无理由退款\n订单里选不想要了即可"
}
```

上面会最多发两条文字：

```text
7天内支持无理由退款
订单里选不想要了即可
```

## 图文组合

图片必须作为文字规则的补充，优先写进动作规则。

```json
{
  "enabled": true,
  "name": "会员专区：权益目录图文",
  "keywords": ["会员权益", "包含什么内容", "课程目录"],
  "actions": [
    {
      "type": "text",
      "text": "目前有专属视频、直播回放、社群和专区问答\n您可以先看课程目录"
    },
    {
      "type": "image",
      "path": "config/reply-images/image3.jpg"
    }
  ]
}
```

## 忽略规则

客户道谢后不再继续补话。

```json
{
  "enabled": true,
  "name": "感谢结束：不再补话",
  "keywords": ["谢谢", "感谢", "明白了", "知道了", "ok", "OK"],
  "actions": [
    {
      "type": "ignore"
    }
  ]
}
```

## 可用动作

- `text`：发文字。字段：`text`。
- `image`：发本地图片。字段：`path`。
- `file`：发本地文件。字段：`path`。当前没有默认文件规则，后续需要时再补。
- `product`：发商品或邀请下单。字段：`productId`、`button`。`button` 可填 `发商品` 或 `邀请下单`。
- `material`：发素材库内容。字段：`subtab`、`materialName` 或 `query`、`button`。当前没有默认素材规则。
- `quick_reply`：发快捷语。字段：`quickReply` 或 `query`。当前没有默认快捷回复规则。
- `ignore`：命中后不发消息，只标记已处理。

## 校验要求

保存前点击“校验规则库”。

校验会检查：

- 每条规则是否有关键词
- 商品动作是否写了 `productId`
- 商品动作是否写了 `button`
- 图片和文件是否写了路径
- 素材库和快捷回复是否写了匹配字段
- `ignore` 动作是否可直接通过
