# 小店AI客服运行状态说明

> 本文定位：主控台、悬浮窗和通知共用的运行状态词典。上级规范：[ARCHITECTURE.md](../ARCHITECTURE.md)，UI 演进见 [workbench-optimization-plan.md](workbench-optimization-plan.md)。

主控制台和桌面悬浮窗使用同一个运行状态源。每个短状态最多 6 个字符，用于快速判断程序正在做什么；状态下方的详细说明不受 6 字限制，用来解释当前客户、动作和失败原因。

## 指示灯

| 颜色 | 含义 | 常见场景 |
| --- | --- | --- |
| 蓝色 | 正在处理 | 检测消息、匹配规则、AI 思考、发送内容 |
| 绿色 | 正常或完成 | 检测中、文字已发、图片已发、商品已发 |
| 黄色 | 等待或暂停 | 等待消息、等待 AI、待扫码、暂停中 |
| 红色 | 失败或超时 | 回复失败、回复超时、脚本失联 |

## 检测与等待

| 短状态 | 说明 |
| --- | --- |
| `启动中` | 正在启动主进程、客服页和自动回复服务 |
| `脚本就绪` | 自动回复脚本已经注入客服页 |
| `检测中` | Bot 已开启，持续监听客户消息 |
| `检测消息` | 正在读取当前会话的最后一条消息 |
| `已检测消息` | 已找到一条需要判断的客户消息 |
| `客服最后` | 当前会话最后一条消息来自客服，不重复回复 |
| `等待消息` | 当前没有可处理的新会话 |
| `暂无消息` | 当前会话还没有客户消息 |

## 匹配与 AI

| 短状态 | 说明 |
| --- | --- |
| `匹配规则` | 正在匹配文字规则或组合动作规则 |
| `匹配图片` | 已匹配到图片回复规则 |
| `匹配商品` | 已匹配到商品卡片或邀请下单规则 |
| `收集上下文` | 正在读取最近消息和客服页上下文 |
| `查询判断库` | 正在查询本地缓存和外部判断库 |
| `AI思考中` | AI 正在结合上下文和判断库生成回复 |
| `AI已返回` | AI 已经生成可发送的内容 |
| `等待AI` | 已发送承接语，继续等待详细答案 |

## 发送过程

| 短状态 | 说明 |
| --- | --- |
| `正在回复` | 回复流程已经接管当前客户消息 |
| `发送文字` | 正在填写并发送文字 |
| `发送图片` | 正在上传并发送图片 |
| `发送商品` | 正在选择并发送商品卡片 |
| `邀请下单` | 正在选择商品并邀请客户下单 |
| `发送文件` | 正在上传并发送文件 |
| `发送素材` | 正在从素材库选择内容 |
| `发送承接` | AI 超过 15 秒，正在生成并发送承接语 |
| `发送兜底` | 60 秒内仍无答案，正在生成兜底回复 |

## 完成结果

| 短状态 | 说明 |
| --- | --- |
| `文字已发` | 文字回复已经发送成功 |
| `图片已发` | 图片回复已经发送成功 |
| `商品已发` | 商品卡片已经发送成功 |
| `已邀下单` | 邀请下单已经发送成功 |
| `文件已发` | 文件已经发送成功 |
| `素材已发` | 素材库内容已经发送成功 |
| `快捷已发` | 后台快捷语已经发送成功 |
| `回复完成` | 当前组合动作已经全部完成 |

## 控制、登录与异常

| 短状态 | 说明 |
| --- | --- |
| `暂停中` | Bot 已暂停，只监控，不自动发送 |
| `跳过重复` | 当前会话已经发送过相同内容 |
| `已忽略` | 规则要求忽略本条消息 |
| `待扫码` | 微信小店客服页需要扫码登录 |
| `等二维码` | 正在等待二维码完整显示后再通知 |
| `页面加载` | 客服页正在打开或重新加载 |
| `后台运行` | 主窗口已隐藏，程序仍在后台运行 |
| `回复超时` | 客户消息超过设定时间仍未完成回复 |
| `回复失败` | 本次回复动作未完成，详细说明会显示原因 |

## 两个界面的显示范围

- 主控制台“总览状态”显示当前步骤、详细说明、服务状态和当前回复来源。
- 展开悬浮窗显示当前步骤、详细说明、AI/本地/脚本/登录指示灯、实时钟和两个主要操作。
- 最小化悬浮窗显示当前短状态、实时摘要和指示灯，并保留打开控制台、展开、关闭三个按钮。
- 悬浮窗关闭后，程序不会退出；可在主控制台“悬浮窗”页重新打开。

## 自动化验证

运行以下命令会启动独立测试实例，依次模拟 18 个关键状态，并检查控制台、展开悬浮窗和最小化悬浮窗是否同步、是否溢出、短状态是否超过 6 个字符，以及最小化态三个按钮是否完整：

```bash
npm run test:status-ui
```
