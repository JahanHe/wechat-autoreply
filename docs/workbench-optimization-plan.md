# 小店AI客服新版工作台优化方案

> 本文定位：工作台信息架构和 UI 演进专题规范。上级规范：[ARCHITECTURE.md](../ARCHITECTURE.md)，全局菜单规范：[desktop-native-menu-guidelines.md](desktop-native-menu-guidelines.md)。

这份方案来自 v0.4.0 前的工作台重构讨论，并在 v0.4.1 开始落地为新版主控台。它用于约束后续继续把主控台从“功能配置集合”优化成“对象、状态、动作、反馈”的高频工作台。

> 后续桌面端全局入口以 [桌面端原生菜单设计规范](./desktop-native-menu-guidelines.md) 为准。该规范要求 Mac 使用系统顶部菜单栏，Windows 使用小型三条杠菜单，并移除当前主窗口内固定 64px 顶部栏。

## v0.4.6 实现状态

| 模块 | 状态 |
| --- | --- |
| 侧边栏 | 已收敛为四个一级入口、悬浮窗最小化态入口和左侧常驻详情 |
| 顶部栏 | 已移除：由 Mac 原生菜单栏、Windows 三条杠菜单、侧边栏状态和页面局部工具栏承接 |
| 上下文栏 | 已承载二级导航和当前页面上下文 |
| 主工作区 | 已保留原业务页面和表单，降低功能回归风险 |
| 详情面板 | 已迁入侧边栏底部，支持规则摘要、日志追踪、异常队列和 Webhook 队列 |
| BrowserView | 已不再因旧顶部栏固定下移，客服页映射由侧边栏和上下文栏边界约束 |
| 客户记忆 | 已进入知识库二级页，可搜索、展开、压缩和追踪未完成事项 |
| 监控看板 | 已从文字卡片改为视觉状态卡、来源比例条和右侧快照详情 |

## 目标

新版工作台要让客服或运营在 5 秒内知道三件事：

| 问题 | 页面应给出的答案 |
| --- | --- |
| 现在发生了什么 | 当前步骤、登录、脚本、AI API、中转服务和异常状态 |
| 哪里需要处理 | 待处理异常、失败回复、外部同步失效、索引异常、Webhook 队列 |
| 我能做什么 | 暂停/开启 Bot、重载客服页、打开悬浮窗、测试规则、修复配置 |

工作台不是营销页，不需要大 Hero、卖点卡片或长文案。它应该是稳定、紧凑、可扫描、可重复使用的操作界面。

## 整体结构

v0.4.1 的已落地结构如下，后续菜单改造时需要移除固定顶部栏：

```text
┌──────────────────────────────────────────────┐
│ 侧边栏       │ 上下文栏                    │
│ 一级导航/状态  ├───────────────────────────────┤
│              │ 主工作区                    │
│              │ 客服页 / 规则列表 / 日志 / 设置 │
│              ├───────────────────────────────┤
│              │ 状态 / 动作反馈               │
└──────────────┴───────────────────────────────┘
```

需要查看对象摘要时，使用左侧常驻详情面板：

```text
┌──────────────┬──────────────────────────────┐
│ 侧边栏       │ 主工作区                    │
│ 导航/状态/详情 │ 列表 / 表格 / 客服页          │
└──────────────┴──────────────────────────────┘
```

## 信息架构

一级导航保持四个入口，侧边栏展开态只显示图标和名称，折叠态只显示图标：

| 一级入口 | 主对象 | 主动作 |
| --- | --- | --- |
| 工作台 | 微信小店客服页和当前会话 | 暂停/开启 Bot、重载页面、捕捉结构 |
| 知识库 | 知识总览、规则、AI 参考、测试中心、客户记忆 | 查看资料、编辑规则、模拟完整回复路径、压缩客户记忆 |
| 监控 | 当前状态、日志、异常、Webhook 队列 | 刷新、检查 AI、测试 Webhook、查看追踪记录 |
| 设置 | 初始化、通知、悬浮窗、开机启动、退出 | 保存设置、自检、彻底退出 |

二级入口只在上下文栏、Windows 三条杠菜单或页面内页签出现，不再让侧边栏承载大量子菜单。上下文栏的二级页签统一放在左侧，不在右侧重复出现，也不显示页面说明小字；页签容器不显示外框，只有当前页签显示选中框。

侧边栏从上到下固定为：Logo、`小店AI客服控制台`、店铺名、悬浮窗状态入口、四个一级入口、工作台详情。未登录时店铺名显示 `微信小店`，Logo 使用本地默认图；客服页登录后如果能从微信小店页面读取店铺 Logo，则替换为店铺 Logo。展开态下 Mac 红黄绿安全区和折叠按钮归属于侧边栏顶部；折叠态下红黄绿和折叠按钮归属于顶部安全区，顶部横跨全窗口，窄侧栏从顶部下方开始且只承载 Logo、悬浮窗灯号和一级入口图标。折叠按钮在展开态和折叠态使用同一位置基准，并和上下文栏二级页签保持同高同层同距；Mac 顶部控件间距固定按 `12px` 基准处理。悬浮窗状态入口复用真实悬浮窗最小化态的灯号、短状态和短摘要，在侧边栏内保留 `展开`、`以最小号打开`、`隐藏` 三个操作。侧边栏不再单独重复展示 Bot、AI、通知状态条。

## 关键页面

### 工作台

- 主区保留微信小店客服原网页映射。
- 侧边栏和运行监控显示当前步骤、AI API、中转服务、脚本、登录和 Bot 状态。
- BrowserView 不再为旧顶部栏额外留出固定高度。
- 高频全局操作通过 Mac 菜单栏、Windows 三条杠菜单、托盘和快捷键触发；页面内只保留和当前对象直接相关的局部操作。

### 知识库

- 二级入口固定为知识总览、规则库、AI 参考、测试中心和客户记忆。
- 知识总览只呈现规则、本机资料、外部同步资料、AI 人格和异常统计；详情提供“前往编辑”入口，不直接堆叠大型表单。
- 规则库按文字、组合动作、图片、文件、商品卡、邀请下单和候选池筛选。规则默认显示紧凑摘要，同一时间只展开一条编辑器。
- 图片和文件动作在展开后显示路径、缩略图、替换按钮和打开位置按钮，收起时不加载预览。
- AI 参考统一查看回复人格、本机自建资料和已经下载到本机的外部同步资料。
- 客户记忆用于查看客户历史、最近消息、客服回复、已发送规则、已执行动作和未完成任务；内容过长时支持本机压缩和 AI 压缩。
- 测试中心区分智能路由、规则匹配、规则动作、AI 本机资料、AI 全部本机资料和知识检索。默认只模拟，真实动作必须二次确认。

### 监控

- 默认展示当前状态、最近回复、异常队列和 Webhook 队列。
- 单条日志打开左侧详情，显示规则命中、本机知识命中、AI 思考状态、发送动作和失败原因。
- 正常技术状态默认折叠；异常自动展开并给出修复入口。

### 设置

- 首次初始化只在缺配置时自动出现。
- 初始化后入口放在 `系统设置 > 初始化与修复`。
- Bot 策略、AI API、外部知识同步、Webhook、悬浮窗、开机启动、防休眠等低频配置按分区折叠。
- 外部知识同步只负责授权、下载和增量更新；生产回复始终读取本机索引。
- 危险区保留“彻底退出程序”，必须二次确认。

## 悬浮窗规范

悬浮窗只显示状态，不承担复杂配置。

| 模式 | 内容 |
| --- | --- |
| 展开态 | 当前状态、AI API、中转服务、脚本、登录、实时钟、打开控制台、暂停/开启 Bot |
| 最小化态 | 状态灯、短状态、摘要、打开控制台、展开、隐藏 |

要求：

- 展开态固定尺寸，不允许无级缩放。
- 最小化态保留三个按钮：打开控制台、展开、隐藏。
- 悬浮窗关闭只是隐藏，可从主控台、Dock 或托盘恢复。
- 主控台和悬浮窗读取同一状态快照，标签、颜色、顺序和更新时间一致。

## 视觉规则

- 使用浅色主调：暖白背景、白色面板、珊瑚主操作色、语义状态色。
- 卡片圆角 8px，不做卡片套卡片。
- 状态不是按钮，操作才是按钮。
- 图标必须配文字；复杂说明放 tooltip 或右侧说明面板。
- 不使用大面积深色背景、装饰渐变、营销式 Hero。
- 表格、列表、状态行优先，适合长时间值守。

## 数据和接口

保持现有 IPC 兼容，后续可以新增只读聚合接口：

```text
main-get-workbench-snapshot
window.mainShell.getWorkbenchSnapshot()
```

建议快照包含：

- `status`
- `settingsHealth`
- `recordsSummary`
- `judgmentStatus`（表示外部同步状态，不表示生产远程查询）
- `notifyOutbox`
- `healthIssues`
- `lastTrace`
- `currentView`

前端再把快照转成 ViewModel：`statusLamps`、`ruleItems`、`logItems`、`settingSections`、`detailPanelItem`。

## 验收标准

- 主控台入口不重复、不遗漏。
- 用户 5 秒内能看到当前状态、异常和下一步动作。
- 客服页映射不遮挡侧边栏、上下文栏和页面局部操作。
- 悬浮窗展开态和最小化态均无遮挡、无文字截断。
- 规则、日志、异常、外部同步资料和 Webhook 都能打开详情。
- `1440x900`、`1280x800`、`390x844` 无横向溢出。
- `test:status-ui`、`test:regressions`、`test:lifecycle`、`doctor` 通过。
