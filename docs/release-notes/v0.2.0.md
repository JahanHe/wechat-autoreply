# 小店AI客服 v0.2.0

> 这一版把“悬浮窗里做所有事”升级为“主控制台 + 客服页映射 + 悬浮窗辅助”。下载后仍只需要补 DeepSeek API Key、企业微信 Webhook，并扫码登录微信小店客服。

## 下载文件

| 系统 | 文件 | 说明 |
| --- | --- | --- |
| macOS Apple Silicon | `wechat-autoreply-macos-arm64.dmg` | M 系列 Mac |
| Windows 安装版 | `wechat-autoreply-windows-setup.exe` | 正常安装到 Windows |
| Windows 便携版 | `wechat-autoreply-windows-portable.exe` | 免安装直接运行 |

## 主要变化

- 新增主控制台：客服页映射、总览状态、Bot、API、Webhook、规则库、日志库、悬浮窗和说明页。
- 客服页从主窗口直接加载改为 BrowserView 映射，原微信小店页面保持独立登录态。
- 悬浮窗隐藏不再退出程序，可从主控制台或系统托盘重新打开。
- 规则库改为可视化卡片：文字、图片、商品卡片、邀请下单、文件、素材库、快捷语和忽略都可编辑。
- 回复日志增加来源分类：动作规则库、文本规则库、图片规则库、直接承接、等待补偿、AI 接管。
- Webhook 支持事件规则、成功回复推送模式、小时总结间隔、每日总览时间和汇总明细条数。
- API 页支持 Key、Base URL、模型、思考模式、推理强度、审核、超时和真实回复测试。
- 扫码登录截图会等待二维码出现后发送；BrowserView 截图不可用时自动截主窗口客服区域。
- 修复本地代理导致 Webhook HTTP 4xx/5xx 被误判成功的问题。

## 首次运行

| 步骤 | 位置 | 操作 |
| --- | --- | --- |
| 1 | 主控制台 > API 接入 | 填 DeepSeek API Key，保存后点“生成测试回复” |
| 2 | 主控制台 > Webhook | 填企业微信机器人 Webhook，保存后点测试 |
| 3 | 主控制台 > 客服页映射 | 扫码登录微信小店客服，并选中一个会话 |
| 4 | 主控制台 > 规则库 | 按需要调整文字、图片、商品、邀请下单规则 |
| 5 | 主控制台 > 日志库 | 查看每次回复的来源、状态和失败原因 |

- `npm run doctor`：生产就绪检查通过。

- `npm run dist:mac`：本地 macOS DMG 构建通过。

## 仍需登录后确认

本次构建时客服页停留在扫码页，未完成登录后的真实会话回归。需要登录后再确认：

- 选中真实会话后发送图片。
- 发送商品卡片。
- 邀请下单。
- 发送文件。
- 查看日志库中对应来源和状态。

## 安全边界

- 不要提交或转发 `.env`。
- 真实 DeepSeek API Key 和企业微信 Webhook 只保存在本机运行目录。
- 本版本不会绕过扫码、验证码或平台风控。
- 严禁引导客户加微信、打电话、私聊、留手机号或私下交易。
