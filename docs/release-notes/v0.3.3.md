# 小店AI客服 v0.3.3

> 这一版修复外部判断库接入在换电脑后容易 404 的问题，并补齐“错误已恢复”的 Webhook 通知闭环。

## 下载

| 系统 | 文件 | 说明 |
| --- | --- | --- |
| macOS Apple Silicon | `wechat-autoreply-macos-arm64.dmg` | 拖到 Applications 后打开 |
| Windows 安装版 | `wechat-autoreply-windows-setup.exe` | 正常安装到 Windows |
| Windows 便携版 | `wechat-autoreply-windows-portable.exe` | 不安装，双击运行 |

## 主要变化

- 修复 Runyu 判断库 Base URL：即使误粘完整 `/api/sync/judgments/query`，程序也会自动规整为 `https://runyuai.zhiduoke.com.cn`，避免重复拼路径导致 404。
- 修复 Cookie 导入：支持裸 token、`session_token=...`、完整 `Cookie: ...`，程序只保存和发送 `session_token`。
- 判断库请求增加 `curl --resolve --noproxy` 备用线路，和 Runyu 接入指南一致，减少 DNS/代理导致的误报 404。
- 判断库 401 提示更明确：需要重新登录 Runyu，并从 Chrome Application > Cookies 复制 `session_token`，不要用 Session Storage。
- Webhook 增加恢复通知：AI、判断库、客服页、自动回复脚本、扫码登录、控制台和通知通道发生故障后，恢复成功会再发一条“已恢复”通知。
- 客服页映射右下角旧版“客服Bot / 悬浮窗 / 设置 / 结构”工具条已移除；这些入口统一放在主控制台和桌面悬浮窗。

## 判断库排查

| 状态 | 含义 | 处理 |
| --- | --- | --- |
| 200 | 接入成功 | 可以测试 10 条或下载本地库 |
| 401 | Cookie 过期或来源不对 | 重新登录 Runyu，从 Cookies 面板复制 `session_token` |
| 403 | 当前账号无权限 | 申请对应判断库权限 |
| 404 | 地址或网络解析问题 | Base URL 只填域名；本版会自动规整并走备用线路 |
