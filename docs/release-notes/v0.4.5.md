# 小店AI客服 v0.4.5

> `v0.4.5` 重构知识体系、规则编辑和测试入口。生产回复不再实时访问外部知识源，而是统一检索已经下载到本机的资料。

## 重点变化

| 模块 | 变化 |
| --- | --- |
| 知识体系 | 明确区分直接执行的本地规则库和供 AI 参考的本机知识资料 |
| 外部同步 | 外部连接只允许授权、下载和增量同步，Cookie 只供同步任务使用 |
| 生产回复 | `/reply` 只读取本机索引，Trace 固定记录 `remoteRequestMade: false` |
| 统一索引 | knowledge-base、手动知识、参考回复和外部同步缓存共用本机检索层 |
| 规则库 | 规则改为紧凑摘要行，同一时间只展开一条编辑器 |
| 测试中心 | 新增智能路由、规则匹配、规则动作、AI 本机资料、AI 全部本机资料和知识检索六种模式 |
| 设置 | Bot 策略、AI API 和外部知识同步从知识库移到设置 |

## 安全边界

- 正常客服回复不会访问外部知识源域名。
- 外部同步失败时继续使用最后一次成功下载的资料。
- Cookie 过期只影响更新，不影响本机规则、AI API 和已有资料检索。
- 模拟测试不会发送客户消息、写客户记忆或生成规则候选。
- 真实规则动作必须先选择客服会话，并通过二次确认。

## 自动测试

- `test:local-knowledge`：验证本机文件、手动资料和外部同步资料进入统一索引。
- `test:production-local-only`：启动伪 AI 与伪外部服务，确认 `/reply` 对外部资料源请求数为 0。
- `test:status-ui`：覆盖知识总览、紧凑规则编辑、测试中心和悬浮窗截图。
- `test:regressions`：继续覆盖图片预览、商品码、非文本消息、异步最终回复和访问凭证恢复。

## 兼容说明

- 原有 `rules`、`actionRules`、`imageReplies` 不需要重新录入。
- 原 `useRemote` 配置迁移为“允许同步联网”，不再表示生产回复远端查询。
- 外部同步缓存仍独立存储，不会改写用户的 knowledge-base 文件。
- 内部历史字段 `judgment*` 暂时保留以兼容旧日志与安装数据，用户界面统一显示“外部同步资料”。

## 真实环境验收

发布前需在真实微信小店会话验证文字、图片、文件、商品卡和邀请下单动作，并在全新用户目录完成首次初始化、外部资料下载和本机索引检索。GitHub Actions 负责构建 Windows 两种资产并执行启动冒烟测试；macOS DMG 在本机执行资源完整性和启动检查。
