# 小店AI客服 v0.4.1

> `v0.4.1` 是新版工作台 UI 落地版。它在 `v0.4.0` 开源发布基线上，把主控台真正升级为 Sidebar、Top Bar、Context Bar、Main Workspace 和 Detail Panel 的工作台结构。

## 下载文件

| 系统 | 文件 |
| --- | --- |
| macOS Apple Silicon | `xiaodian-ai-kefu-macos-arm64.dmg` |
| Windows 安装版 | `xiaodian-ai-kefu-windows-setup.exe` |
| Windows 便携版 | `xiaodian-ai-kefu-windows-portable.exe` |

## 这版解决什么

| 方向 | 结果 |
| --- | --- |
| 主控台结构 | Sidebar 只保留一级入口，Top Bar 承载状态和高频动作，Context Bar 承载二级导航 |
| 客服页映射 | BrowserView 下移，给 Top Bar 留出空间，客服页不再覆盖全局控制入口 |
| 详情面板 | 规则卡片和回复日志可在右侧查看摘要、动作、AI Trace、判断库和失败原因 |
| 工作台快照 | 新增只读 `main-get-workbench-snapshot` IPC，聚合状态、配置健康、日志、异常和通知队列 |
| 系统设置 | 保留现有配置能力，并在工作台详情里显示异常队列和 Webhook 队列 |
| 悬浮窗 | 保留 v0.4.0 展开态和 mini 三按钮逻辑，只同步状态顺序和文案 |

## 使用边界

- 微信小店客服页是第三方网页映射和自动化对象，请只在自己的店铺、账号权限和平台规则范围内使用。
- Runyu 判断库是私有外部服务，需要授权账号和权限；本项目不会提供或绕过权限。
- DeepSeek API Key、Webhook、Cookie、控制 Token 只应保存在本机，不应进入仓库、日志、截图或安装包。

## 真实环境验收

本版发布前的自动门禁覆盖：

- `build-extension`
- `test:status-ui`
- `test:regressions`
- `test:lifecycle`
- `test:release-readiness`
- `check:secrets`
- `doctor`
- `test:packaged-resources`
- `test:macos-package`

安装后的真实微信小店会话仍建议确认：

| 场景 | 预期 |
| --- | --- |
| 客服页映射 | 微信小店页面显示在工作区，顶部状态栏和全局操作不被覆盖 |
| 规则库 | 手动测试、真实执行、图片预览、替换和打开位置仍可用 |
| 日志 Trace | 点击日志后右侧详情显示规则命中、AI Thinking、判断库、动作和失败原因 |
| 悬浮窗 | 展开态和 mini 态仍能打开控制台、暂停/开启 Bot、隐藏和恢复 |
| 生命周期 | 主窗口关闭后 Bot 继续运行，Dock/托盘/悬浮窗可恢复，彻底退出需二次确认 |
