# 小店AI客服 v0.3.4

> 本版将外部知识库从“手工复制访问凭证”升级为“每台电脑网页登录一次，应用自动接入并持续恢复”。

## 下载

| 系统 | 文件 | 说明 |
| --- | --- | --- |
| macOS Apple Silicon | `wechat-autoreply-macos-arm64.dmg` | 拖到 Applications 后打开 |
| Windows 安装版 | `wechat-autoreply-windows-setup.exe` | 正常安装到 Windows |
| Windows 便携版 | `wechat-autoreply-windows-portable.exe` | 不安装，双击运行 |

## 主要变化

- 初始化页和外部知识库页增加“打开网络配置页”“获取访问凭证”“重新配置”“清除本机凭证”。
- 外部知识库使用独立持久会话 `persist:runyu-auth`，不会与微信小店登录状态混用。
- 登录后自动读取 `session_token`，写入该电脑的本地配置，并执行真实外部知识库查询。
- 只有真实查询成功才显示“已连接”；区分等待登录、验证中、凭证过期、账号无权限和连接异常。
- 应用重启时自动从本机会话恢复 Cookie 并再次验证，不需要每次手工复制。
- 保留 访问凭证 输入框作为自动登录失败时的备用方式。

## 本机真实验证

- 外部知识库配置窗口成功打开并完成登录。
- 自动读取凭证并写入 `~/Library/Application Support/小店AI客服/.env`。
- “会员”真实查询成功，远端返回 94 条记录并生成本地缓存。
- 关闭并重新启动应用后，没有要求再次登录，自动恢复会话并再次查询成功。
- 配置窗口在验证成功后自动关闭。

## 状态说明

| 状态 | 含义 | 处理 |
| --- | --- | --- |
| 已连接 | 凭证和权限均通过真实查询 | 可以正常查询和更新缓存 |
| 登录已过期 | 服务端返回 401 | 点击“重新配置” |
| 账号无权限 | 服务端返回 403 | 为当前账号开通对应工具权限 |
| 连接异常 | 网络或接口异常 | 检查网络、Base URL 后重新测试 |
