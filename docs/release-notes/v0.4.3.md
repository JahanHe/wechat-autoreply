# 小店AI客服 v0.4.3

> `v0.4.3` 是 macOS 安装修复和运行恢复维护版。它不改变自动回复核心逻辑，重点让 DMG 安装遇到拦截时可以双击脚本自动处理，并补齐本地运行恢复链路。

> 当前架构和菜单演进以 [ARCHITECTURE.md](../../ARCHITECTURE.md) 和 [桌面端原生菜单设计规范](../desktop-native-menu-guidelines.md) 为准。

## 这版解决什么

| 方向 | 结果 |
| --- | --- |
| macOS 安装 | DMG 根目录新增 `repair-macos-install.command`，遇到“无法验证开发者”“App 已损坏或无法打开”等问题时可以双击运行 |
| 安装修复 | 脚本会复制或覆盖修复 `/Applications/小店AI客服.app`、检查 `Info.plist` 和 `app.asar`、清除隔离属性并打开应用 |
| 发布门禁 | macOS 安装包冒烟测试和发布就绪检查会验证 DMG 内必须包含可执行的一键修复脚本 |
| 运行恢复 | 启动时同步运行目录 Webhook 配置，修复安装后配置存在但自检误报的问题 |
| 外部知识库 | 新访问凭证会自动验证，成功后关闭配置窗口，并保留过期、无权限、错误和超时状态 |
| 客服页桥接 | 配置存储桥迁移到 `window.wechatKfDesktop.storage`，避免覆盖客服页自己的 `window.chrome` |
| 登录截图 | Webhook 登录二维码截图优先截取已挂载的客服页 `BrowserView` 区域 |

## 使用边界

- 微信小店客服页是第三方网页映射和自动化对象，请只在自己的店铺、账号权限和平台规则范围内使用。
- 外部知识库是私有外部服务，需要授权账号和权限；本项目不会提供或绕过权限。
- DeepSeek API Key、Webhook、访问凭证、控制 Token 只应保存在本机，不应进入仓库、日志、截图或安装包。
- `repair-macos-install.command` 只处理本机安装和 macOS 隔离属性，不上传任何密钥、登录态或运行缓存。

## 真实环境验收

本版发布前的自动门禁覆盖：

- `build-extension`
- `test:release-readiness`
- `check:secrets`
- `doctor`
- `test:packaged-resources`
- `dist:mac`
- `test:macos-package`

安装后的真实微信小店会话仍建议确认：

| 场景 | 预期 |
| --- | --- |
| macOS 首次打开 | 正常拖入 Applications 后可打开；如被拦截，双击 DMG 内一键修复脚本后可再次打开 |
| Webhook 配置 | 运行目录已有配置时，doctor、自检和通知配置应保持一致 |
| 外部知识库配置 | 获取新访问凭证后自动验证，成功后配置窗口自动关闭 |
| 客服页登录 | 登录二维码截图通知截取客服页区域，不截错主控台其它位置 |
| 悬浮窗 | 展开态和 mini 态仍能打开控制台、暂停/开启 Bot、隐藏和恢复 |
