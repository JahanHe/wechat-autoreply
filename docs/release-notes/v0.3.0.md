# 微信小店客服自动回复 v0.3.0

> 这一版重点补齐“下载安装后默认可用”：默认动作规则、图片/商品/邀请下单、判断库、浅色控制台、状态悬浮窗、应用图标和构建发布流程都进入正式桌面版。

## 下载文件

| 系统 | 文件 | 说明 |
| --- | --- | --- |
| macOS Apple Silicon | `wechat-autoreply-macos-arm64.dmg` | M 系列 Mac |
| Windows 安装版 | `wechat-autoreply-windows-setup.exe` | 正常安装到 Windows |
| Windows 便携版 | `wechat-autoreply-windows-portable.exe` | 免安装直接运行 |

## 主要变化

- 新增 Runyu 判断库模块：支持 Cookie 配置、来源/类型选择、测试 10 条、全部下载本地库、进度条和定时增量刷新。
- 回复策略改为 AI/判断库优先：15 秒无结果才轮换发送慢回复承接语，60 秒无可用答案才轮换发送兜底回复。
- 慢回复承接语和兜底回复都支持列表轮换，不再固定重复一句。
- 默认注入脚本内置会员专区动作规则，即使首次配置覆盖失败，也能识别权益、进群、发商品和邀请下单意图。
- 同一会话记忆已发送文字，避免同一个客户反复收到相同静态话术。
- 收窄联系方式规则，避免“手机号登录”“本地私域引流”这类问题被误判成索要联系方式。
- 规则库图片/文件路径支持“选择/替换”和“打开位置”，不再只能看路径文本。
- 主控制台统一浅色设计；悬浮窗改为状态灯，隐藏后可从控制台重新打开。
- 应用图标、托盘图标和安装包图标统一使用 `build/icon.*`。

## 默认动作规则

| 场景 | 默认动作 |
| --- | --- |
| 会员专区包含什么权益 | 文字加课程目录图 |
| 会员专区怎么使用、怎么进群 | 文字加两张说明图 |
| 想买会员、会员链接、发商品链接 | 发年度会员商品卡片 |
| 怎么买、怎么付款、怎么下单 | 邀请下单 |
| 月度会员取消自动续费 | 文字加图 |
| 咨询俱乐部详情 | 文字加图 |

默认商品码：`10000275472384`。

## 首次运行

| 步骤 | 位置 | 操作 |
| --- | --- | --- |
| 1 | 主控制台 > API 接入 | 填 DeepSeek API Key，保存后点“生成测试回复” |
| 2 | 主控制台 > Webhook | 填企业微信机器人 Webhook，保存后点测试 |
| 3 | 主控制台 > 判断库 | 可选填 Runyu Cookie，测试 10 条或下载本地库 |
| 4 | 主控制台 > 客服页映射 | 微信扫码登录并选中测试会话 |
| 5 | 主控制台 > 规则库 | 按需修改文字、图片、文件、商品和邀请下单规则 |

## 已验证

- `node --check`：主进程、控制台、预加载脚本、注入脚本、AI 服务。
- 默认规则静态匹配：权益、进群、发商品、邀请下单都命中动作规则。
- AI/判断库测试：“本地私域引流可以吗？”返回判断库/AI 答案，不再触发联系方式固定回复。
- 默认图片文件存在：`config/reply-images/image1.png` 到 `image5.png`。
- 本机运行配置迁移到 `desktop-0.3.0`，15 秒承接和 60 秒兜底阈值生效。

## 仍需登录后确认

如果客服页停留在扫码页，需要扫码后再做真实会话回归：

- 发送权益图文。
- 发送进群说明图文。
- 发送年度会员商品卡片。
- 邀请下单。
- 发送文件动作。

## 安全边界

- 不提交 `.env`、DeepSeek API Key、Webhook 或 Runyu Cookie。
- 工具不会绕过扫码、验证码或平台风控。
- 不引导客户加微信、打电话、私聊、留手机号或私下交易。
