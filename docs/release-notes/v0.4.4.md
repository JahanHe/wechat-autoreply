# 小店AI客服 v0.4.4

> `v0.4.4` 是回复链路重构版。它统一“本机回复中转服务 / AI API 回复”术语，并给每条客户消息建立可追踪的回复任务、客户记忆、完成度检查和规则候选池。

## 重点变化

| 模块 | 变化 |
| --- | --- |
| 术语 | “本地 AI 服务”统一改为“本机回复中转服务”，真实生成内容的部分统一叫“AI API” |
| 回复任务 | 每条客户消息生成 `ReplyTask`，记录客户、消息、路径、承接、最终回复、完成度和补救次数 |
| 规则模式 | 规则支持 `final`、`quick_then_api`、`action_only`、`ignore` 四种模式，旧规则默认 `final` |
| 15 秒承接 | 承接语只表示正在处理，不再把任务标记为最终完成 |
| 60 秒延迟 | 60 秒默认只标记延迟、写日志和通知，不直接给客户发送固定兜底 |
| 完成度检查 | 新增 `/completion/check`，由本机回复中转服务调用 AI API 或启发式检查回复是否解决问题 |
| 持续补救 | `partial` / `unknown` 可自动补救，客户可见补发最多 2 次，超限转人工 |
| 客户记忆 | 按会话生成客户 ID，记录最近消息、已发规则、已发动作、未完成任务和完成度 |
| 规则候选池 | AI API 成功回复后生成候选规则，必须人工审核后才能进入正式规则库 |
| 日志和 Webhook | 回复记录和总结新增任务 ID、客户 ID、规则模式、完成度、补救次数、延迟任务和规则候选统计 |

## 兼容说明

- 旧规则不需要立刻迁移；没有 `mode` 字段时默认按 `final` 执行。
- 图片、文件、商品卡片、邀请下单仍走原客服页自动化动作，只在外层增加任务追踪和完成度检查。
- `AI_SERVICE_NAME` 仍保持 `xiaodian-ai-service`，用于安装包健康检查兼容；用户可见名称已改为“本机回复中转服务”。
- API Key、Webhook、判断库访问凭证、控制 Token 和运行缓存仍只保存在本机运行目录，不进入仓库或安装包。

## 真实环境验收

- 新客户首次咨询会生成客户记忆和回复任务。
- 本地规则命中后进入回复完成度检查。
- `quick_then_api` 会先本地快速回复，再补 AI API 最终回复。
- 未命中规则时，15 秒无最终答案会发送承接语，AI API 返回后继续补发最终答案。
- 60 秒无最终答案只记录延迟和通知，不默认给客户发固定兜底。
- 完成度未完成时最多自动补救 2 次，之后转人工并记录原因。
- 日志详情能看到 taskId、customerId、规则模式、是否发承接、是否发最终回复、完成度和补救次数。
