> 以 `v0.3.7` 为功能基线重构。`v0.3.8` 的界面方案已废弃，保留其中验证有效的安全和生命周期修复。

## 这版解决什么

| 方向 | 结果 |
| --- | --- |
| 后端结构 | 主进程拆出 AppContext、状态中心、IPC 契约、配置校验、安全控制、AI/知识库和扩展构建链路 |
| 回复能力 | 继续保护文字、图片、文件、商品卡片、邀请下单、规则库、AI、判断库和异步第二条回复 |
| 判断库 | DeepSeek 改原生 fetch，判断库保留 fetch 主线和受控 curl 备用线；日志记录查询词、来源、命中、耗时和线路 |
| 配置安全 | 本机控制 Token、CORS 白名单、配置修复备份和敏感值脱敏 |
| 客服页脚本 | 使用 esbuild 生成 IIFE，非文本消息、图片/商品/文件动作和异步回复有独立状态 |
| 退出闭环 | macOS 关闭主窗口只隐藏，Dock 可恢复；危险区提供二次确认的彻底退出 |
| 界面 | 四个一级入口；悬浮窗提供固定展开态和三按钮最小化态，关闭后可从主控台、Dock 或托盘恢复 |

## 首次使用

1. 安装后打开“小店AI客服”。
2. 进入 `系统设置 > 初始化`，填写 DeepSeek API Key 和企业微信 Webhook。
3. 点击“打开登录网页”，完成 Runyu 判断库网页登录，再点“我已登录，获取凭证”。
4. 点击“保存并自检”。
5. 进入 `客服工作台` 扫码登录微信小店客服，并选中测试会话。

真实 Key、Webhook、Cookie 和本机控制 Token 只写入本机运行目录，不进入仓库或安装包。

- `check:secrets`
- `doctor`
- `test:release-readiness`

真实微信小店会话需要在安装后的应用里确认：

| 场景 | 预期 |
| --- | --- |
| 文字规则 | 能命中会员、权益、进群、购买等默认规则 |
| AI 和判断库 | 复杂问题先走 AI/判断库，15 秒承接、60 秒兜底，最终答案不被承接语取消 |
| 非文本消息 | 客户图片、表情、商品卡、文件和视频不会导致脚本失联 |
| 页面动作 | 能发送图片、文件、商品卡片和邀请下单，并记录成功/失败原因 |
| 故障恢复 | 登录过期、判断库失效、AI 失败和 Webhook 失败有错误通知，恢复后有恢复通知 |
| 窗口生命周期 | 主窗口隐藏、Dock 恢复、悬浮窗重开、开机启动和彻底退出语义一致 |
