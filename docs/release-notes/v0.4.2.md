# 小店AI客服 v0.4.2

> `v0.4.2` 是外部知识库命名和发布链路修正版。它不改变自动回复核心逻辑，重点统一主控台、文档、测试门禁和安装包里的模块命名。

> 当前架构和菜单演进以 [ARCHITECTURE.md](../../ARCHITECTURE.md) 和 [桌面端原生菜单设计规范](../desktop-native-menu-guidelines.md) 为准。

## 下载文件

| 系统 | 文件 |
| --- | --- |
| macOS Apple Silicon | `xiaodian-ai-kefu-macos-arm64.dmg` |
| Windows 安装版 | `xiaodian-ai-kefu-windows-setup.exe` |

## 这版解决什么

| 方向 | 结果 |
| --- | --- |
| 模块命名 | 主控台和文档统一使用“外部知识库”，不再使用旧模块名或特定服务名 |
| 接入方式 | 明确外部知识库支持网络调用，也可以导入本地数据后检索 |
| 安全边界 | API Key、Webhook、访问凭证、控制 Token 和运行缓存继续只保存在本机 |
| 发布链路 | README、发布说明、发布就绪检查、安装包资源检查和冒烟脚本同步到 `0.4.2` |
| GitHub 链接 | 当前文档入口统一指向规范仓库 `JahanHe/Shop-ai-reply` |

## 使用边界

- 微信小店客服页是第三方网页映射和自动化对象，请只在自己的店铺、账号权限和平台规则范围内使用。
- 外部知识库是私有外部服务，需要授权账号和权限；本项目不会提供或绕过权限。
- DeepSeek API Key、Webhook、访问凭证、控制 Token 只应保存在本机，不应进入仓库、日志、截图或安装包。

## 真实环境验收

本版发布前的自动门禁覆盖：

- `build-extension`
- `test:status-ui`
- `test:desktop-modules`
- `test:lifecycle`
- `test:release-readiness`
- `check:secrets`
- `doctor`
- `test:packaged-resources`
- `test:macos-package`

安装后的真实微信小店会话仍建议确认：

| 场景 | 预期 |
| --- | --- |
| 外部知识库配置 | 网络配置页、获取访问凭证、检查连通性、初始化本地缓存文案一致 |
| 本地导入 | 未使用网络凭证时，外部知识库仍可按本地缓存或导入数据说明理解 |
| 日志追踪 | 点击日志后显示规则命中、AI 思考状态、外部知识库、动作和失败原因 |
| 悬浮窗 | 展开态和 mini 态仍能打开控制台、暂停/开启 Bot、隐藏和恢复 |
| 生命周期 | 主窗口关闭后 Bot 继续运行，Dock/托盘/悬浮窗可恢复，彻底退出需二次确认 |
