# 小店AI客服 v0.4.0

> `v0.4.0` 是稳定开源基线版本。当前推荐使用 `v0.4.4`，它在此基础上实现新版工作台 UI，并补齐 macOS 安装一键修复脚本。

## 下载文件

| 系统 | 文件 |
| --- | --- |
| macOS Apple Silicon | `xiaodian-ai-kefu-macos-arm64.dmg` |
| Windows 安装版 | `xiaodian-ai-kefu-windows-setup.exe` |
| Windows 便携版 | `xiaodian-ai-kefu-windows-portable.exe` |

## 这版解决什么

| 方向 | 结果 |
| --- | --- |
| 规则匹配 | 真实客服会话和手动激发测试共用同一套归一化、分词和匹配逻辑 |
| 默认动作 | 继续保护文字、图片、文件、商品卡片、邀请下单、规则库、AI、外部知识库和异步第二条回复 |
| 悬浮窗 | 最小化态保留打开控制台、展开、隐藏三个按钮；展开态保持固定尺寸和状态同步 |
| 安装包 | macOS 和 Windows 安装资产加入资源完整性与启动冒烟测试 |
| CI | 发布工作流使用 Node 24 兼容的 Actions，并在发布前运行发布就绪检查 |
| 开源材料 | 新增 README、CHANGELOG、LICENSE、CONTRIBUTING，明确使用边界和贡献规则 |

## 使用边界

- 微信小店客服页是第三方网页映射和自动化对象，请只在自己的店铺、账号权限和平台规则范围内使用。
- 外部知识库是私有外部服务，需要授权账号和权限；本项目不会提供或绕过权限。
- DeepSeek API Key、Webhook、访问凭证、控制 Token 只应保存在本机，不应进入仓库、日志、截图或安装包。

## 首次使用

1. 安装后打开“小店AI客服”。
2. 进入 `系统设置 > 初始化`，填写 DeepSeek API Key 和企业微信 Webhook。
3. 点击“打开网络配置页”，完成 外部知识库网页登录，再点“获取访问凭证”。
4. 点击“保存并自检”。
5. 进入 `客服工作台` 扫码登录微信小店客服，并选中测试会话。

## 真实环境验收

本版发布前的自动门禁覆盖：

- `build-extension`
- `test:extension-modules`
- `test:status-ui`
- `test:release-readiness`
- `check:secrets`
- `doctor`
- `test:packaged-resources`
- `test:macos-package`
- `test:windows-packages`

安装后的真实微信小店会话仍建议确认：

| 场景 | 预期 |
| --- | --- |
| 文字规则 | 能命中会员、权益、进群、购买等默认规则 |
| 图片规则 | 客户问会员专区、课程目录、进群等问题时能发送对应图片 |
| 商品动作 | 能发送商品卡片和邀请下单，并匹配商品码 |
| AI 和外部知识库 | 复杂问题先走 AI/外部知识库，15 秒承接、60 秒兜底，最终答案不被承接语取消 |
| 非文本消息 | 客户图片、表情、商品卡、文件和视频不会导致脚本失联 |
| 故障恢复 | 凭证过期、外部知识库失效、AI 失败和 Webhook 失败有错误通知，恢复后有恢复通知 |
| 窗口生命周期 | 主窗口隐藏、Dock 恢复、悬浮窗重开、开机启动和彻底退出语义一致 |
