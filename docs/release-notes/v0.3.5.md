# 小店AI客服 v0.3.5

> 本版把自动回复过程改成可追踪的统一状态机。检测、规则匹配、外部知识库、AI 思考、文字、图片、商品、文件和异常过程会同步显示在主控制台与悬浮窗。

## 下载

| 系统 | 文件 | 说明 |
| --- | --- | --- |
| macOS Apple Silicon | `wechat-autoreply-macos-arm64.dmg` | 拖到 Applications 后打开 |
| Windows 安装版 | `wechat-autoreply-windows-setup.exe` | 正常安装到 Windows |
| Windows 便携版 | `wechat-autoreply-windows-portable.exe` | 不安装，双击运行 |

## 主要变化

- 新增统一运行状态机，所有短状态最多 6 个字符。
- 总览状态新增“当前步骤”和“最近步骤”，保留最近 6 次状态变化。
- 悬浮窗新增当前步骤、详细说明、类别、红绿灯和最近 3 次状态。
- 最小化悬浮窗只保留指示灯和当前短状态，不再出现文字遮挡。
- 脚本心跳与业务步骤拆分：AI 思考、发送图片时不会把脚本误判为异常。
- 文字、图片、商品卡片、邀请下单、文件、素材和快捷语都有独立的发送中与已完成状态。
- AI 流程细分为收集上下文、查询外部知识库、AI 思考、AI 返回、等待 AI 和发送回复。
- 新增 `npm run test:status-ui`，自动验证 14 个关键状态、双窗口同步、六字限制和窗口溢出。
- 《项目历程》新增第二段英雄之旅，完整记录 `v0.1.0` 到 `v0.3.5` 从可交付安装包走向可值守桌面工作台的升级过程。

## 代表状态

| 阶段 | 短状态 |
| --- | --- |
| 消息检测 | `检测中`、`检测消息`、`已检测消息`、`客服最后` |
| 规则和 AI | `匹配规则`、`查询外部知识库`、`AI思考中`、`AI已返回` |
| 正在发送 | `发送文字`、`发送图片`、`发送商品`、`邀请下单` |
| 发送完成 | `文字已发`、`图片已发`、`商品已发`、`已邀下单` |
| 控制和异常 | `暂停中`、`跳过重复`、`回复超时`、`回复失败` |

完整说明：[运行状态说明](../runtime-statuses.md)

## 验证

- 主进程、控制台、悬浮窗和客服注入脚本语法检查通过。
- 14 个关键状态通过正式 `bot-status` IPC 状态机同步到主控制台和悬浮窗。
- 展开悬浮窗 `320 x 252` 无内容溢出。
- 最小化悬浮窗 `210 x 44` 无文字遮挡。
- 所有正式短状态均不超过 6 个字符。
